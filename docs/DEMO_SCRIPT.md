# Demo Script (Milestone 7)

A suggested 5-6 minute walkthrough covering every milestone. Record your screen (e.g. OBS,
Loom, or Windows/Mac built-in recorder) while narrating these beats.

1. **Login & 2FA (Milestone 6)** — Log in as an entrepreneur. Show the 2FA code arriving as a
   toast, enter it, land on the dashboard. Mention the password strength meter on the
   Register page.
2. **Dashboard tour (Milestone 1 & 7)** — Point out the theme consistency (colors, cards,
   layout), then click "Take a Tour" in the navbar and click through the 5 steps.
3. **Meetings (Milestone 2)** — Add an availability slot, switch account (or narrate) to show
   requesting a meeting against that slot, accept it, show it appear on the calendar and on
   the dashboard's "Upcoming Meetings" card.
4. **Video Call (Milestone 3)** — From a confirmed meeting, click "Join Call". Start the call
   to show the live camera preview, toggle mic/camera, demo screen share, then end the call.
5. **Document Chamber (Milestone 4)** — Upload a PDF, change its status to "In Review", open
   the signature pad and sign it, show the status badge flip to "Signed".
6. **Payments (Milestone 5)** — As an investor, show the wallet balance, deposit funds, then
   use "Fund a Deal" to send money to an entrepreneur; switch views to show the transaction
   appear in both wallets' histories.
7. **Role-based access (Milestone 6)** — Try navigating to the other role's dashboard URL
   directly to show it redirects back automatically.
8. **Wrap-up** — Show the sidebar once more, highlighting that every new module (Meetings,
   Documents, Payments) is reachable from navigation, then mention responsiveness by resizing
   the browser or opening dev tools' device toolbar.

## Before recording
- `npm run build && npm run preview` to record against the production build rather than the
  dev server, for a cleaner demo.
- Clear `localStorage` (or use a private/incognito window) beforehand so the seeded demo data
  looks intentional rather than showing leftover test entries from earlier testing.
